### Code Convention

- 변수명 : snake 표기법을 사용한다
- 함수
	- 함수는 소문자로 하고, 카멜케이스 표기로 한다.
	- getter, setter 함수는 멤버 변수와 이름을 일치시킨다.
	- 생성자 및 setter의 파라미터 이름은 멤버변수와 동일하게 사용한다.

- 괄호
	- 한 줄로 쓸 수 있는 코드 (주로 getter) 는 한 줄에 쓰기

- 클래스 정보 초기화
	- 생성자 호출 시에는 객체 생성만 담당하고, 따로 init()를 두어 초기값 할당을 수행한다. (생성자 호출 중 예외 객체를 던지면 소멸자 호출 안되는 걸 방지하기 위함)

- RAII 원칙
	- 해당 클래스 내의 멤버 변수는 소멸자 내에서 모두 정리되어야 하며, 해당 클래스 멤버변수의 해제를 다른 클래스에 위임해서는 안된다.
